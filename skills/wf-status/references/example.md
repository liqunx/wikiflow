## 常见场景

### 场景 1：每日进度查看

每天开始工作前，查看所有计划的进度：

```bash
/wf-status
```

**用途**：
- 快速了解所有活跃计划
- 决定今天要处理哪个计划
- 识别长期未更新的计划

### 场景 2：决策前查看

在决定下一步工作前，查看活跃计划：

```bash
/wf-status --active
```

**用途**：
- 只看活跃的计划
- 决定哪个需要提交测试
- 决定哪个需要修复 bug

### 场景 3：详细分析

需要详细了解某个计划时：

```bash
/wf-status <计划名> --detail
```

**用途**：
- 查看完整的任务列表
- 识别阻塞的任务
- 评估剩余工作量

### 场景 4：问题排查

查找有问题（阻塞）的计划：

```bash
/wf-status --blocked
```

**用途**：
- 找到所有阻塞的计划
- 分析阻塞原因
- 决定优先处理顺序

## 输出格式说明

### 进度条

进度条使用 ASCII 字符显示：

- `█`：已完成的部分
- `░`：未完成的部分
- 总共 10 格

**示例**：
```
0%:   ░░░░░░░░░
10%:  █░░░░░░░░░
23%:  ██░░░░░░░░
50%:  ██████░░░░
75%:  ████████░░
100%: ██████████
```

### 任务统计

格式：`(已完成/总数 任务完成)`

**示例**：
- `(24/65 任务完成)`：24 个已完成，总共 65 个
- `(0/15 任务完成)`：刚创建，还没有任务完成
- `(10/10 任务完成)`：全部完成

### 状态标签

- `active`：活跃开发中
- `draft`：草稿，刚创建
- `paused`：暂停
- `blocked`：阻塞
- `completed`：已完成（但未归档）

## 注意事项

### 只读操作

`/wf-status` 是只读命令，不会修改任何文件：
- ✅ 安全：不会意外修改数据
- ✅ 快速：不需要写权限
- ✅ 可随时使用：即使在执行其他任务时

### 目录要求

需要在文档仓库中执行：
- 必须存在 `.wikiflow/config.json`
- 必须存在 `docs/changes/active/` 目录
- 每个计划必须包含 `spec.md` 和 `tasks.md`

### 性能考虑

- 扫描目录：O(n)，n 为计划数量
- 解析文件：O(m)，m 为任务数量
- 适合中小型项目（< 100 个计划）

## 故障排查

### 问题：没有显示任何计划

**可能原因**：
1. `docs/changes/active/` 目录不存在
2. 目录为空
3. 所有计划都缺少必需的文件

**解决方案**：
```bash
# 检查目录是否存在
ls docs/changes/active/

# 如果不存在，创建一个计划
/wf-plan "新功能名称"

# 如果缺少文件，检查目录结构
ls docs/changes/active/<计划名>/
```

### 问题：显示警告信息

**警告示例**：
```
⚠️ 警告：计划 2026-04-24-xxx 缺少 spec.md，跳过
```

**解决方案**：
1. 检查文件是否存在
2. 如果文件缺失，重新生成计划
3. 或手动删除不完整的目录

### 问题：进度显示不正确

**可能原因**：
1. `tasks.md` 格式不正确
2. 任务标记使用了错误的格式

**正确格式**：
```markdown
- [ ] 待执行任务
- [x] 已完成任务
```

**错误格式**：
```markdown
* [ ] 待执行任务  # 使用了 * 而不是 -
- [] 待执行任务   # 缺少空格
```

## 高级用法

### 定制输出格式

虽然 `/wf-status` 提供了标准化的输出，但你可以通过以下方式定制：

1. **使用筛选**：只显示需要的计划
2. **使用详细视图**：查看完整信息
3. **结合其他命令**：
   ```bash
   # 保存状态到文件
   /wf-status > status.txt

   # 搜索特定内容
   /wf-status | grep "阻塞"
   ```

### 自动化脚本

可以在脚本中使用 `/wf-status`：

```bash
#!/bin/bash
# 每日检查脚本

# 获取状态
/wf-status --active

# 检查是否有阻塞的计划
if /wf-status --blocked | grep -q "blocked"; then
  echo "⚠️ 发现阻塞的计划！"
fi
```
