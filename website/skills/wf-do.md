# wf-do - 执行开发

按照 spec.md 和 tasks.md 逐个执行开发任务。

## 使用方式

```bash
# 执行当前 active 的计划
/wf-do

# 执行指定的计划
/wf-do docs/changes/active/2026-04-24-add-registration

# 执行并跳过已完成的任务
/wf-do --continue
```

## 执行流程

### 1. 前置检查

- 读取 `.wikiflow/config.json` 获取配置
- 扫描 `docs/changes/active/` 目录
- 如果有多个 active 变更，要求用户选择

### 2. 读取计划

- **spec.md**：了解目标、方案、影响范围
- **tasks.md**：获取任务清单

### 3. 执行任务

```
FOR 每个未完成的任务:
  1. 读取任务描述
  2. 根据 project.code.root 定位代码目录
  3. 执行任务（修改代码）
  4. 标记任务为已完成（更新 tasks.md）
```

### 4. 输出进度

```
📊 开发进度

计划：添加注册功能
总任务：8
已完成：3 / 8（37.5%）

✅ [后端] 设计 users 表结构
✅ [后端] 实现注册 API
✅ [前端] 创建注册页面
⬜ [后端] 实现登录 API 更新
⬜ [后端] 编写单元测试
⬜ [前端] 更新登录页面
⬜ [前端] 移除首次改密弹窗
⬜ [前端] 添加表单验证
```

## 与 /wf-fix 的交互

开发过程中发现问题：

```bash
场景：执行到"创建注册页面"时，发现应该"注册后自动登录"

选项：
1. 先完成当前任务，然后 /wf-fix  ✅ 推荐
2. 暂停当前任务，立即 /wf-fix
3. 忽略，继续按计划执行
```

## 注意事项

- 每完成一个任务就更新 `tasks.md`（标记为已完成）
- 如果需要修改 specs，提示用户执行 `/wf-fix`
- 不自动修改 `spec.md`（避免意外改变计划）
- 任务执行失败时，不自动跳过，等待用户确认

## 完成提示

```bash
✅ 所有任务已完成

下一步：
  → /wf-fix 修正问题（如果有）
  → /wf-finish 编译到 Wiki
```

## 下一步

- 所有任务完成后，使用 `/wf-finish` 编译到 Wiki
- 如果发现问题，使用 `/wf-fix` 修正
