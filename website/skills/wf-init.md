# wf-init - 初始化项目

初始化 WikiFlow 项目，在用户项目中创建文档驱动的开发环境。

## 使用方式

```bash
# 初始化当前项目
/wf-init

# 初始化指定项目
/wf-init /path/to/project
```

## 功能说明

### 1. 对话确认配置

向用户询问以下信息：

- **Q**: 项目名称是什么？
- **Q**: 代码在哪个目录？（如 `src/`、`lib/`、`app/`）→ 用于 AI 定位代码
- **Q**: 主要编程语言是什么？
- **Q**: 工作流使用什么语言？（默认 `zh-CN`）
- **Q**: 文档放在哪个目录？（默认 `docs/`）

### 2. 检查现有目录

检查项目根目录下是否已有 `docs` 目录：

- **不存在**：直接创建
- **已存在**：询问用户选择处理方式

### 3. 创建目录结构

```
{docs}/
├── wiki/
│   ├── current/          ← 当前实现（AI 查询优先读这里）
│   ├── history/          ← 历史版本
│   ├── deprecated/       ← 已废弃功能
│   ├── decisions/        ← 决策日志
│   ├── index.md          ← 主索引
│   └── log.md            ← 变更日志
├── raw/
│   └── 00-uncategorized/ ← 原始文档
└── changes/
    └── active/           ← 开发中的变更
```

### 4. 生成配置文件

- `index.md` - Wiki 主索引
- `log.md` - 变更日志
- `.wikiflow/config.json` - 项目配置

## 注意事项

- 如果用户项目已有文档，建议先放入 `raw/` 目录，后续通过工作流整理
- `project.code` 是面向 AI 的配置，用户无需关心代码细节
- `config.json` 可随时手动编辑，修改后立即生效
- 重复执行 `/wf-init` 会检测到已有配置，询问是否覆盖

## 下一步

初始化完成后，可以使用：

- `/wf-plan "功能描述"` - 开始规划新功能
- `/wf-search "关键词"` - 查询现有文档
