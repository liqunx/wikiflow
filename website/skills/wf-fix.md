# wf-fix - 修正问题

修正开发过程中的问题和需求改进。AI 自动判断是 bug 还是需求改进，更新 spec.md 和 tasks.md。

## 使用方式

```bash
# 修正 Bug
/wf-fix "注册成功后应该自动登录"

# 需求改进
/wf-fix "用户反馈：注册流程太复杂，需要简化"
```

## 功能说明

### 1. 自动判断类型

AI 根据描述自动判断：

- **Bug 修复**：与原计划不符的错误行为
- **需求改进**：新的功能需求或优化建议

### 2. 更新文档

- **spec.md**：添加问题描述、影响分析、解决方案
- **tasks.md**：添加新的修复任务
- **decisions.md**：记录重要决策（如果适用）

### 3. 生成决策日志

对于重要的变更，会生成决策记录：

```markdown
# 2026-04-24-auto-login-after-registration

## 问题描述
用户注册后需要手动登录，体验不佳。

## 决策
注册成功后自动登录用户，无需手动输入密码。

## 原因
- 减少用户操作步骤
- 符合业界最佳实践
- 提升用户体验

## 影响
- 需要修改注册 API 返回 token
- 前端需要处理自动登录逻辑
```

## 使用场景

### Bug 修复

```bash
/wf-fix "用户反馈：注册成功后仍然停留在注册页面"
→ AI 判断：这是 Bug
→ 更新 tasks.md：添加修复任务
→ 提示：继续 /wf-do 执行修复
```

### 需求改进

```bash
/wf-fix "产品经理要求：添加手机号注册功能"
→ AI 判断：这是需求改进
→ 更新 spec.md：添加新功能说明
→ 更新 tasks.md：添加新任务
→ 提示：需要重新评估影响范围
```

### 问题澄清

```bash
/wf-fix "注册时邮箱验证格式不正确"
→ AI 分析：这是 Bug 还是需求理解偏差？
→ 询问：请提供更多信息
→ 根据回答决定处理方式
```

## 注意事项

- 仅在 `/wf-do` 之后使用（需要 active 变更）
- AI 会自动判断问题类型，调整处理方式
- 重要变更会生成决策日志
- 可能会重新评估任务优先级

## 下一步

- 修正后，继续使用 `/wf-do` 执行任务
- 如果影响较大，可能需要重新执行 `/wf-plan`
